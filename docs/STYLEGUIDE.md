# Docs Style Guide

ドキュメントは **読み手がすぐ動ける実用性** を最優先とします。

## 原則
- 事実と手順を先に。背景や考察は必要に応じて章立てで分離。
- 「どのワークフローを走らせ／どのURLで確認するか」を必ず書く。
- 例は **コピー&ペースト可能** な形で。
- UI 表記・用語は **HANDBOOK / Glossary** と統一。

## 記法
- 時刻は **JST** を明記（例: JST 04:20）。
- 日付は `YYYY-MM-DD` 固定。
- コードブロックは言語を指定（`bash`, `yaml`, `json`, `js`, `clojure` など）。
- ファイル/パスは先頭に `/` を付けてリポ直下からの相対に統一（例: `/public/daily/...`）。

## PR / レビュー
- 大きな変更は **E2E / Docs / Ops** をワンセット。
- 破壊的変更は **Release checklist** を埋めてからマージ。
- 外部リンクは **link-check (docs)** が通ること。

## スクリーンショット / ログ
- Actions Summary やブラウザコンソールの抜粋は引用ブロックで貼る。
- 画像は本文で要点を説明（「どこを見れば良いか」）。
