# v1.3 Release Notes (2025-09-03)

## Summary
v1.3 では、初期表示の JS 負荷削減と計測の健全化を中心に、**体感速度と計測安定性**を底上げしました。E2E は全緑を維持しつつ、LHCI の主要警告（MPFID/TTI/Bootup/Main-thread work）を解消しています。

## Changes
- **Lighthouse Budgets** を微引き締め: total ≤ 2.2MB / script ≤ 1.2MB / requests ≤ 160
- **Lazy import**
  - `boot_auto.mjs` を導入し、`?auto=1`/`?daily_auto=1` 時のみ auto系を読込
  - `media_player.mjs` はメディア問題の時だけ動的読込
  - `levenshtein` を `fuzzy.mjs` に分離し回答時のみ動的読込
- **Off-main-thread JSON parse（Worker）**
  - `dataset.json` / `aliases*.json` の `JSON.parse` を Worker に移動
  - Alias マップ構築は **小分け yield**（50件単位）で長いタスクを回避
  - 起動時の alias ロードは停止し、**Start 後に非同期ロード**
- **LHCI 配線の是正**
  - `.lighthouserc.budgets.json` の `collect.settings.budgetsPath` を設定
  - 計測URLを `/app/?lhci=1` に変更し、解析スクリプトを計測から除外
  - `uses-long-cache-ttl` は WARN 監視（上限12件）に方針化（ops-runbook に記載）

## Outcomes
- **E2E (matrix)**: 緑（share含む）
- **LHCI**: MPFID/TBT/Bootup/Main-thread work の主要警告が解消。残る `uses-long-cache-ttl` は Pages 既定/外部由来のため WARN 管理。
- **運用性**: `asset-scan` ワークフローを追加（on-demand/PR）。未使用候補は現状 0 件。

## Notes for Operators
- 計測時は `/app/?lhci=1` を使用（`index.html` の解析スクリプト guard に一致）
- 退行監視は Budgets のみ厳格、TTL は WARN のみで十分（詳細: `docs/ops-runbook.md`）
